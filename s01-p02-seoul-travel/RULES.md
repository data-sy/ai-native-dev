# Seoul Travel Project - Data Rules

이 파일은 서울 여행 데이터 프로젝트의 데이터 일관성 규칙을 정의합니다.

## 데이터 일관성 규칙

### 1. admission_fee (입장료)
**필수 규칙: 모든 장소의 `admission_fee`는 반드시 원화(KRW) 단위를 포함하여 문자열로 명시한다.**

- ✅ 올바른 예시:
  - `"성인 3,000 KRW"`
  - `"1일권 성인 62,000 KRW"`
  - `"무료"`
  - `"무료 (상설전시 기준, 기획전시 별도)"`
  - `"없음"`
  - `"없음 (클럽 입장료 별도)"`

- ❌ 잘못된 예시:
  - `3000` (숫자만 있음)
  - `"3000"` (단위 없음)
  - `null` (명시적 문자열로 표현 필요)

### 2. tags (태그)
- 배열 형식으로 작성
- 최소 3개 이상 권장
- 해시태그 형식 사용: `["#전통미", "#인생샷", "#한복대여"]`

### 3. operating_hours (운영시간)
- 명시적인 시간 문자열 또는 `null`
- 예시: `"09:00 - 18:00 (화요일 휴궁)"`

### 4. recommended_course (추천 코스)
- `course_title`: 코스 컨셉 제목
- `summary`: 한 문장 요약
- `plan`: 3단계 시간 계획 배열 (맛집 포함)

## 자연어 쿼리 → JSON 변환 패턴

사용자의 자연어 검색 요청을 다음 JSON 구조로 변환합니다:

```json
{
  "city": "도시명",
  "tags": ["태그1", "태그2"],
  "period_days": 숫자 또는 null,
  "sort_by": "정렬방식",
  "limit": 숫자 또는 null
}
```

### 변환 규칙

**period_days (기간):**
- "어제" → 1
- "최근 일주일" → 7
- "최근 한 달" → 30
- 명시 없음 → null

**sort_by (정렬):**
- "인기 많은", "인기있는" → "popularity"
- "최신", "최근" → "latest"
- "평점 높은" → "rating"
- 명시 없음 → "latest" (기본값)

**limit (개수 제한):**
- "3개만", "5곳만" → 숫자 추출
- "모든", "전체" → null
- 명시 없음 → null

## 파일 구조

- `seoul_spots_final.json`: 최종 완성된 서울 명소 데이터
- `seoul_hansik_restaurant_query.json`: 자연어 쿼리 변환 예시
- `seoul-images/`: 10개 명소 이미지
- `travel_info.txt`: 추가 여행 정보 (병합 소스)
